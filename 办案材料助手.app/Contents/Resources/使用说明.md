# 办案材料助手使用说明

这是默认优先本机处理、也可主动选择在线 OCR 后端的办案材料转换工具。双击应用可打开图形界面，也可把文件直接拖到应用图标上转换。

## 可处理内容

| 输入文件 | 输出 |
| --- | --- |
| PDF、PNG、JPG、TIFF、HEIC 等图片 | 可选本地 MinerU、本机 Apple Vision OCR，或 legal-ocr 在线后端生成 Markdown |
| Word、Excel、CSV、PPT、HTML 等 | 生成 Markdown |
| Markdown、TXT | 生成 Word 文档 |
| 音频、视频 | 生成 Markdown 逐字稿 |
| 微信聊天录屏 MP4/MOV/M4V | 生成取证 PDF 初稿、截图和复核资料 |

`.doc` 老格式会先由 macOS 转成 `.docx` 后再提取内容。PDF/图片可以在界面中选择不同引擎：`MinerU（本地）` 适合版面和表格，`VisionOCR（Apple Vision）` 适合快速离线初稿，`legal-ocr` 选项会调用本机安装的 skill，再按配置走 PaddleOCR/MinerU 在线后端或轻量接口并执行保守法律后处理。表格会尽量输出为 Markdown 表格；复杂扫描件仍需对照原件复核。

## 输出与归档

日常输出目录：

```text
~/Desktop/VisionOCR_Output/
```

每次转换还会在以下目录保存原件副本、转换结果和处理记录：

```text
~/Desktop/VisionOCR_Output/archive/时间戳_文件名/
├── input/
├── output/
└── metadata.json
```

归档用于复核来源与结果，不表示 Markdown 可以还原扫描件的原始版面。

## 使用方式

双击：

```text
/Applications/办案工具集/办案材料助手.app
```

界面中的“转 Markdown”页面支持 PDF、图片、Word、Excel 及音视频；“Markdown → Word”页面用于输出 Word；“微信录屏取证”页面用于把聊天录屏筛成清晰、少重复、可复核的 PDF 初稿。“转 Markdown”和“微信录屏取证”页面都有“证据整理”按钮，用于从当前输出文件夹重新读取 Markdown、PDF、OCR 报告和索引，生成案件材料汇总、时间线初稿和案件OS输入包。

微信录屏取证默认输出到：

```text
~/Desktop/录屏取证输出/
```

快速初稿全程本地处理；OCR增强默认只走本机 OCR。只有在页面内勾选“手动云端整理”时，才会发送 OCR 文字、时间戳、截图编号和 SHA256，不发送录屏或截图图像，且结果需人工核实。

## 证据整理

“证据整理”不会读取程序刚生成时的缓存，而是每次都重新扫描当前输出文件夹。因此，如果你先打开 Markdown 或报告手动修改，再点击“证据整理”，整理结果会以修改后的当前文件为准。

整理结果会写入当前输出目录下的：

```text
证据整理/
├── 案件材料汇总.md
├── 案件时间线.md
├── materials_manifest.json
└── case_os_intake_package.json
```

录屏取证页面点击“证据整理”时，会优先整理最近一次取证导出目录。如果该目录只有取证 PDF、没有 Markdown，工具会自动补转一份 Markdown 放在“证据整理/自动补转MD/”中，再纳入本次整理。原始 PDF、截图、Markdown 和 OCR 报告不会被删除或覆盖。

录屏取证的“每 N 秒留 1 张”默认按秒数严格取图。例如 2 分 29 秒视频，每 1 秒约 149 张，每 5 秒约 30 张。如果勾选“开头详情页加密”，前 8 秒会额外按更密间隔保留，用来避免漏掉聊天详情页，因此总页数会比严格秒数更多。

脚本或 AI 调用：

```bash
/Applications/办案工具集/办案材料助手.app/Contents/MacOS/droplet "/path/to/file"
```

Agent 快速模式：

```bash
/Applications/办案工具集/办案材料助手.app/Contents/MacOS/droplet --agent "/path/to/file"
```

指定 PDF/图片引擎：

```bash
/Applications/办案工具集/办案材料助手.app/Contents/MacOS/droplet --engine mineru "/path/to/file.pdf"
/Applications/办案工具集/办案材料助手.app/Contents/MacOS/droplet --engine visionocr "/path/to/file.pdf"
/Applications/办案工具集/办案材料助手.app/Contents/MacOS/droplet --engine legalocr "/path/to/file.pdf"
/Applications/办案工具集/办案材料助手.app/Contents/MacOS/droplet --engine legalocr-paddle "/path/to/file.pdf"
/Applications/办案工具集/办案材料助手.app/Contents/MacOS/droplet --engine legalocr-mineru "/path/to/file.pdf"
/Applications/办案工具集/办案材料助手.app/Contents/MacOS/droplet --check-engines
```

底层快速入口：

```bash
visionocr-agent "/path/to/file" --output "/path/to/output.md"
visionocr-agent --engine legalocr-mineru "/path/to/file.pdf"
visionocr-agent --check-engines
```

直接打开微信录屏取证页：

```bash
/Applications/办案工具集/办案材料助手.app/Contents/MacOS/droplet --wechat
```

常用参数：

```bash
visionocr-agent "/path/to/file.pdf" --page 1-5
visionocr-agent "/path/to/table.png" --layout tables
visionocr-agent "/path/to/file.pdf" --recognition-level accurate
```

查看本说明：

```bash
/Applications/办案工具集/办案材料助手.app/Contents/MacOS/droplet --help
```

## 注意事项

- 本地 MinerU 与 VisionOCR 不上传材料；`legal-ocr` 选项会按其配置调用 PaddleOCR/MinerU 在线后端或 MinerU 轻量接口，适合需要更好版面/法律后处理效果时主动选择。
- 对扫描材料、图片和 OCR 结果，应对照原件人工核实后再作为正式材料使用。
- 日志位置：`/tmp/vision-ocr-app.log`。
